# Smart Clip MCP — 技术方案与实现方案

> 版本：v0.1 | 日期：2026-06-21

---

## 一、产品定位

**一句话**：面向 AI Agent 的智能视频剪辑 MCP Server，输入一段长视频 + 剪辑意图，输出精选短视频成片。

**核心价值**：不是又一个 FFmpeg wrapper，而是"剪辑大脑"——用字幕语义分析 + LLM 决策驱动剪辑，KyaniteLabs/mcp-video 只当底层执行引擎。

**MVP 目标场景**：口播、播客、直播回放、访谈、课程等语音驱动型内容 → 自动拆条生成短视频。

### 三层解耦

| 层 | 职责 | 关键依赖 | 可替换性 |
|---|---|---|---|
| **Analyzer 分析层** | 视频内容理解：字幕提取、场景检测、音频分析 | Whisper / PySceneDetect / librosa | 可切换本地模型或云端 API |
| **Planner 决策层** | 剪辑策略生成：精彩片段识别、片段排序、模板选择 | LLM (OpenAI/本地) + Prompt 模板 | 可切换 LLM 提供商 |
| **Executor 执行层** | 剪辑执行：裁切、拼接、字幕烧录、格式适配 | mcp-video (KyaniteLabs) / FFmpeg | 可替换为其他执行引擎 |

## 三、核心数据流

```
输入: video_path + intent (自然语言) + config (可选参数)
  │
  ▼
[1] Analyzer.analyze(video_path)
  │ ├─ whisper_transcribe()     → SRT (带时间戳的字幕)
  │ ├─ detect_scenes()          → 场景切换时间点列表
  │ ├─ audio_energy_profile()   → 音频能量曲线 (振幅/秒)
  │ └─ get_metadata()           → 时长/分辨率/帧率/编码
  │
  ▼
[2] Planner.plan(srt, scenes, audio, intent, config)
  │ ├─ segment_by_speech()      → 语义段落 (按气口/句子切分)
  │ ├─ llm_highlight_detect()   → 精彩片段候选 (start, end, reason, score)
  │ ├─ apply_strategy()         → 按策略过滤/排序/合并
  │ └─ generate_edit_plan()     → 结构化剪辑方案 (JSON)
  │
  ▼
[3] Executor.execute(edit_plan, video_path)
  │ ├─ trim_clips()             → 裁切各片段
  │ ├─ merge_with_transitions() → 拼接 + 转场
  │ ├─ add_subtitles()          → 烧录字幕
  │ ├─ add_audio()              → BGM/音效 (可选)
  │ ├─ resize_for_platform()    → 适配目标平台
  │ └─ export()                 → 输出成片
  │
  ▼
输出: { output_path, clips: [...], metadata: {...} }
```

#### 5.2.3 策略引擎

LLM 返回候选片段后，策略引擎做二次加工：

```python
class StrategyEngine:
    """对 LLM 返回的候选片段做策略性调整"""

    def apply(self, candidates: List[ClipCandidate], config: EditConfig) -> EditPlan:
        # 1. 分数过滤: 去掉 weighted_score < threshold 的
        candidates = [c for c in candidates if c.weighted_score >= config.min_score]

        # 2. 去重: 间隔 < 10秒的片段只保留分数更高的
        candidates = self._deduplicate(candidates, min_gap=10)

        # 3. 时长调整:
        #    - 短于 clip_duration_min 的: 向前后扩展至气口处
        #    - 长于 clip_duration_max 的: 在中间气口处切分
        candidates = self._adjust_duration(candidates, config)

        # 4. 数量控制: 取 top-N
        candidates = sorted(candidates, key=lambda c: c.weighted_score, reverse=True)[:config.clip_count]

        # 5. 时间线优化: 调整边界到安全的切割点
        #    - 起点前推 0.3-0.5 秒 (避免开头被吃)
        #    - 终点后推 0.3-0.5 秒 (避免结尾被吃)
        #    - 确保不在说话中间切断
        candidates = self._snap_to_safe_cuts(candidates)

        # 6. 生成剪辑方案
        return EditPlan(
            clips=candidates,
            total_selected_duration=sum(c.duration for c in candidates),
            platform=config.platform,
            with_subtitles=config.with_subtitles,
            with_bgm=config.with_bgm
        )
```

### 8.2 LLM 调用封装

```python
# src/smart_clip/planner/highlight.py
import json
from openai import AsyncOpenAI

class HighlightDetector:
    def __init__(self, model="gpt-4o-mini", temperature=0):
        self.client = AsyncOpenAI()
        self.model = model
        self.temperature = temperature  # 必须为 0，保证可复现

    async def detect(self, segments, intent, config) -> list[ClipCandidate]:
        prompt = self._build_prompt(segments, intent, config)

        response = await self.client.chat.completions.create(
            model=self.model,
            temperature=self.temperature,
            response_format={"type": "json_object"},  # 强制 JSON 输出
            messages=[
                {"role": "system", "content": "你是专业视频剪辑师，擅长识别精彩片段。严格按 JSON 格式输出。"},
                {"role": "user", "content": prompt}
            ]
        )

        result = json.loads(response.choices[0].message.content)
        return [ClipCandidate(**clip) for clip in result["clips"]]

    def _build_prompt(self, segments, intent, config):
        # 将 segments 格式化为带索引的文本
        formatted = "\n".join([
            f"[{i}] ({s.start:.1f}s - {s.end:.1f}s) {s.text}"
            for i, s in enumerate(segments)
        ])
        return HIGHLIGHT_DETECTION_PROMPT.format(
            total_duration=config.total_duration,
            language=config.language,
            intent=intent,
            formatted_segments=formatted,
            clip_duration_min=config.clip_duration_min,
            clip_duration_max=config.clip_duration_max,
            clip_count=config.clip_count
        )
```

### 8.3 字幕安全切割算法

```python
def snap_to_safe_cut(time: float, subtitle: SubtitleResult, direction: str = "forward") -> float:
    """
    将切割时间点对齐到字幕安全位置，避免在说话中间切断

    direction:
        "forward" — 向后找最近的句子结束点 (用于片段起点)
        "backward" — 向前找最近的句子结束点 (用于片段终点)
    """
    # 找到所有句子结束点 (end of segments)
    sentence_ends = [seg.end for seg in subtitle.segments]

    if direction == "forward":
        # 找 >= time 的最近句子结束点
        for end in sentence_ends:
            if end >= time:
                return end
    else:
        # 找 <= time 的最近句子结束点
        for end in reversed(sentence_ends):
            if end <= time:
                return end

    return time  # 兜底: 返回原始时间
```

## 十一、风险与应对

| 风险 | 影响 | 应对 |
|---|---|---|
| LLM 幻觉：编造不存在的片段 | 高 | 输出必须带 segment_index，校验时间码是否在原始字幕范围内 |
| Whisper 中文识别不准 | 中 | 开启 vad_filter + 指定 language="zh"；备选火山引擎 ASR |
| mcp-video 依赖不稳定 | 中 | Executor 同时支持 FFmpeg 直调作为 fallback |
| 纯视觉内容（舞蹈/运动）无字幕信号 | 中 | MVP 限定口播场景；Phase 3 加入音频能量+场景检测联合决策 |
| FFmpeg 本地渲染慢（长视频） | 低 | 支持异步任务模式，先返回 edit_plan，渲染完再通知 |
| KyaniteLabs 大版本升级 breaking change | 低 | 锁定 mcp-video 版本；Executor 层做抽象，可替换 |

## 十二、快速启动验证 (1天内可跑)

在完整开发前，可以先做一个最小验证：

```python
#!/usr/bin/env python3
"""
最小验证脚本: 证明字幕→LLM→剪辑的链路可行
依赖: pip install openai whisper ffmpeg-python
"""
import whisper
import json
from openai import OpenAI

# 1. 转写字幕
model = whisper.load_model("large-v3")
result = model.transcribe("input.mp4", language="zh", word_timestamps=True)

segments = [
    {"index": i, "start": s["start"], "end": s["end"], "text": s["text"]}
    for i, s in enumerate(result["segments"])
]

# 2. LLM 识别精彩片段
client = OpenAI()
response = client.chat.completions.create(
    model="gpt-4o-mini",
    temperature=0,
    response_format={"type": "json_object"},
    messages=[
        {"role": "system", "content": "你是视频剪辑师，严格输出JSON"},
        {"role": "user", "content": f"""从以下字幕中选出5个最精彩的片段:
        {json.dumps(segments, ensure_ascii=False)}

        输出格式: {{"clips": [{{"start": float, "end": float, "reason": str}}]}}"""}
    ]
)

clips = json.loads(response.choices[0].message.content)["clips"]

# 3. FFmpeg 裁切
import subprocess
for i, clip in enumerate(clips):
    subprocess.run([
        "ffmpeg", "-i", "input.mp4",
        "-ss", str(clip["start"]), "-to", str(clip["end"]),
        "-c", "copy", f"output_{i}.mp4"
    ])

print(f"生成 {len(clips)} 个片段!")
```

**跑通这个脚本就能验证核心链路，再决定是否投入完整开发。**
